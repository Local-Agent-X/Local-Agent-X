## System Configuration

- **Embedding Model:** nomic-embed-text (768 dimensions) via Ollama — running locally, zero API cost
- **Search:** Hybrid BM25 keyword + cosine vector similarity (70/30 weight)
- **Chunking:** Conversation-pair (Q+A preserved as semantic units, max ~3200 chars per pair)
- **Storage:** SQLite with FTS5 full-text search + in-memory cosine vector search
- **Additional features:** Structured facts with entity tagging, confidence scoring, temporal metadata, nightly consolidation, memory tiers — not used during benchmark retrieval

### Hardware

- Consumer Windows 11 PC
- Ollama running locally (CPU/GPU inference for embeddings)
- No cloud services, no paid API calls

### Key Differences from MemPalace

Open Agent X matches MemPalace's retrieval performance while providing additional capabilities:

- **Hybrid search** — BM25 keyword + vector similarity (MemPalace uses vector-only)
- **Structured facts pipeline** — entity-tagged facts with confidence scoring, deduplication, and contradiction detection
- **Nightly consolidation** — automatic fact merging, promotion to long-term memory, entity page generation
- **Memory tiers** — hot/warm/cold/archive classification
- **Proactive retention** — agent automatically saves personal and project facts during conversation without user prompting
- **Universal conversation ingest** — supports ChatGPT, Claude.ai, Claude Code, Codex CLI, Slack, and generic JSON exports
- **Conversation-pair chunking** — preserves Q+A semantic units instead of fixed-window splitting

## Methodology

1. Each of the 500 LongMemEval questions includes ~53 "haystack" sessions (distractor conversations) plus 1-2 answer sessions containing the ground truth
2. All sessions are ingested as conversation-pair chunks with metadata (session ID, date)
3. The question is searched against all ingested chunks using hybrid BM25 + vector search
4. A hit is recorded if any of the top-K returned chunks belongs to a ground-truth answer session (matched by session ID)
5. Database is cleaned between questions to prevent cross-contamination

## What Could Push This Higher

1. **LLM reranking** — MemPalace hits 100% with Claude Haiku reranking the top-50 candidates. Adding a rerank pass would likely push us to 98-100% at minimal API cost.
2. **Session grouping** — when a chunk hits, automatically include sibling chunks from the same session for better context coverage.
3. **Two-pass retrieval** — first search finds relevant chunks, extract entities, second search broadens recall.
4. **Temporal-aware scoring** — boost chunks that match date references in the query. Temporal-reasoning (92.5%) is our weakest category and the most improvable.
