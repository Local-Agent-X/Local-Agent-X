## Behavior changes тАФ Alex to decide

The three install scripts were NOT doing different syntax of the same thing. They were doing genuinely different things. The audit said to flag this. Consolidating the COMMON logic into install-common.mjs and shrinking the wrappers to тЙд30 LOC strips the following behaviors:

### Stripped from `install.bat` and `install.ps1` (Windows one-click installer behavior)
- **Web-bootstrap repo download** тАФ old scripts pulled a ZIP from `github.com/petermanrique101-sys/Local-Agent-X` into `%USERPROFILE%\Local-Agent-X`. New scripts assume the repo is already on disk and run from `%~dp0` / `$PSScriptRoot`. **Breaks the `iwr -useb тАж/install.ps1 | iex` one-liner flow.**
- **Git auto-install** тАФ old scripts installed Git via winget if missing. New scripts don't (Git isn't needed for npm/build, and the web-download was the only thing using it).
- **Playwright Chromium install** тАФ old scripts ran `npx playwright install chromium`. New scripts don't. Browser-tool features that rely on Chromium will be broken until the user runs this manually. **You may want to add this to `install-common.mjs`** тАФ it's cross-platform and `install.sh` already omitted it; adding it would expand Linux/Mac install footprint by ~150MB but unify behavior.
- **`npm run build`** тАФ old scripts ran a full build to `dist/`. New scripts don't (dev mode via `npm run dev` doesn't need it; prod users run `npm run build && npm start` per README).
- **Desktop Electron subfolder build** тАФ `install.bat` was running `npx tsc` inside `desktop/`. Gone.
- **Desktop / Start Menu shortcut creation** тАФ `install.bat` and `install.ps1` were creating `.lnk` files and a `launch.vbs` for hidden-window Electron startup. Gone.
- **Auto-launch at end of install** тАФ `install.bat` ran `cscript launch.vbs` at the end; `install.ps1` opened a browser tab at `http://127.0.0.1:7007`. Gone тАФ users now run `npm run dev` manually per the README.

### Stripped from `install.sh` (dev workflow)
- **Ollama service start** (macOS `brew services start ollama` / Linux `nohup ollama serve &`). Gone тАФ install-common assumes Ollama is started by the OS or user separately.
- **Claude Code CLI global install** (`npm install -g @anthropic-ai/claude-code`). Gone. **This is the bigger loss** тАФ your project routes Anthropic OAuth through this CLI on the Max plan. Linux/Mac dev users will hit a dead end when they try to paste a `claude setup-token` and don't have the CLI. **You may want to add this to `install-common.mjs`** тАФ it's cross-platform via `npm i -g` and was missing from `install.bat`/`.ps1` anyway, so consolidating would actually fix a Windows-side gap.

### Added to `install.bat` and `install.ps1` (Linux/Mac parity gained)
- **`~/.sax/settings.json` seed** тАФ was only in `install.sh`. Now in install-common, so Windows users get sensible defaults on first install. (Idempotent тАФ skips if the file already exists.) I think this is a clear win and would not flag it for revert.

## What I think the right call is

1. **Accept the strip** for shortcut/launch/Chromium-build/web-bootstrap тАФ those belong in a separate end-user installer (`install-from-web.ps1` or a packaged Electron installer), not in the dev-install script. Trying to be both is what got the three scripts to drift.
2. **Add Claude Code CLI install** to `install-common.mjs` тАФ it's cross-platform, was already in install.sh, and was a real gap on Windows. Two lines of code.
3. **Add Playwright Chromium install** to `install-common.mjs` IF browser-tool features are on by default. If they're behind a feature flag, leave it out.
